SharedEventStream: 가장 어려운 문제

장기 연결 관리의 현실적 도전

문제

OpenCode SDK의 SSE 클라이언트가 내부적으로 재접속을 시도합니다. 연결이 죽어도 for await 루프가 절대 끝나지 않아 감지 불가능합니다.

해결

  • Liveness timeout (45초) — 이벤트 없으면 강제 재접속
  • Lazy 연결 감지 — 첫 번째 실제 이벤트 수신 후에만 connected 마킹
  • Dual abort controller — 전역 vs 개별 연결 분리
// 연결이 살아있다고 "착각"하는 것을 방지
// markConnected()는 SDK 호출 시점이 아닌
// 첫 번째 실제 이벤트 수신 후에만 실행
stream.subscribe((event) => {
  if (!this.connected) this.markConnected();
  this.dispatch(event);
});
